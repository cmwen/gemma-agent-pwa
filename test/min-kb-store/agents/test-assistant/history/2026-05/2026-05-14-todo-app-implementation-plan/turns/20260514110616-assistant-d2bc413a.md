A basic todo item should include at least a description and a status (e.g., completed/pending).

**Data Structure:** A Todo Item can be represented as an object or dictionary with the following fields:
*   `id`: A unique identifier for the task.
*   `description`: The text description of the task.
*   `is_completed`: A boolean indicating whether the task is finished (defaulting to `False`).

This step involves saving the task list to storage and loading it back when the application starts or needs to be reloaded. For a simple implementation, this can be done using file I/O (e.g., JSON files).

*   **Loading:** A function that reads the data from the file, deserializes it, and populates the in-memory task list with the loaded data.
